# 研发群同步摘要：原型变动

本次原型主要更新两部分，均为前端展示口径调整，便于后续研发统一理解。

## 1. 转诊单详情页流程进度图调整

- 详情页顶部流程图由原来的“按角色 + 事件”的泳道图，调整为“按关键事件展示”的转诊闭环信息。
- 新方案不再强调不同角色各自做了什么，而是统一展示患者这次转诊的主流程进展。
- 闭环统一为 5 个关键节点：
  - 发起转诊
  - 转诊受理
  - 到院/接诊
  - 完成诊疗/接收完成
  - 闭环完成
- 不同角色查看同一张转诊单时，顶部闭环信息结构一致，只允许操作按钮不同。
- 底部“操作日志”继续保留详细过程，用于看系统通知、催办、改派、补充材料等明细动作。
- 研发理解重点：
  - 顶部闭环看“走到哪一步”
  - 底部日志看“过程中做了什么”
  - 两者不要做成重复信息

## 2. CHG-42 相关业务调整

本次 CHG-42 在原型中的核心目标，是让“上转单详情页”按角色展示真正需要的信息，减少无关信息暴露。

### 角色展示口径

- 基层医生 / 基层负责人：
  - 保留完整“到院接诊安排”
  - 保留“转诊预约码”
  - 保留对患者的提醒信息，方便实际通知患者

- 县级经办医生 / 县级科主任：
  - 只看简化状态结果
  - 不展示详细接诊安排
  - 不展示预约码
  - 不展示患者就诊须知

- 转诊管理员：
  - 保留完整到院接诊安排
  - 保留预约码
  - 保留短信预览相关展示

### 信息区块调整

- “到院接诊安排”改为按角色精准显示，不再所有角色都看到完整内容。
- “转诊预约码”不再默认对所有查看角色开放。
- “就诊须知”不再在所有医生角色详情页展示。
- “操作日志”在原型中改为仅管理员查看，避免普通查看角色被大量过程信息干扰。

### F-02 原型展示调整

- 机构配置页新增“患者须知模板”配置展示位。
- 当前用于原型演示，不强调真实接口联动。
- 业务上表达的是：机构可以配置发给患者的补充提示内容。

### 患者短信口径

- 患者短信预览中追加“患者须知模板”内容。
- 业务目标是让短信通知不仅有基础接诊信息，还能附带院方补充提醒。

## 研发落地时需要统一注意

- 这次原型调整重点是“展示口径统一”，不是新增复杂流程。
- 同一张上转单在不同角色下，信息结构应尽量一致，只做必要的可见性裁剪。
- 顶部闭环信息不要再回到泳道图思路。
- 底部操作日志继续承担明细追溯职责。
